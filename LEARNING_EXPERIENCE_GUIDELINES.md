# AI Engineer OS - Learning Experience Specification

## Goal
Every lesson should transform a learner from "I recognize this topic" to "I can explain, build, and use it in production."

## Standard Lesson Flow

1. Hero Section
- Learning objectives
- Prerequisites
- Estimated time
- Difficulty
- XP reward
- Real-world applications

2. Story First
Explain every topic with a memorable story before technical details.

3. Why It Matters
- Enterprise relevance
- Banking example
- AI product example

4. Visual Analogy
Examples:
- RAG = Open Book Exam
- Embeddings = GPS Coordinates
- Vector DB = Smart Library
- Agent = Company CEO
- MCP = Universal USB-C for AI Tools

5. Interactive Animation
Every lesson must include:
- Step-by-step animation
- Play/Pause controls
- Hover explanations
- Speed adjustment

6. Deep Theory
- Beginner explanation
- Intermediate explanation
- Senior explanation
- Architect explanation

7. Interactive Diagram
Users can click each component to reveal detailed explanations.

8. Production Architecture
Show enterprise architecture and data flow.

9. Banking Use Case
Explain how banks use the concept.

10. Code Walkthrough
Python + .NET + TypeScript where relevant.
Each line explained.

11. Common Mistakes
- Beginner pitfalls
- Performance issues
- Security risks
- Cost optimization

12. Interview Prep
- Beginner
- Intermediate
- Senior
- Architect
- System Design

13. Hands-on Lab
A real portfolio project.

14. Quiz
MCQ, drag-drop, architecture, scenario, coding.

15. Flashcards
Auto-generated with spaced repetition.

16. Revision Mode
5-minute summary, one-page cheat sheet, memory hooks.

## AI Tutor
Buttons:
- Explain Simply
- Explain Like a Senior Engineer
- Explain with Banking Example
- Explain with Story
- Explain with Diagram
- Explain in Native Language
- Generate More Examples
- Test My Understanding

## Retention Features
- Analogies
- Mnemonics
- Visual summaries
- Mind maps
- Spaced repetition
- Progress heatmap
- Weekly review

## Video Learning Rules
- Never show broken videos.
- Always provide:
  - Quick overview
  - Deep dive
  - Full lecture
  - Conference talk
- Include AI summary, timestamps, notes, quiz and flashcards.

## Success Criteria
After every lesson the learner should:
- Explain it
- Draw it
- Implement it
- Apply it to Banking
- Pass interview questions
- Teach someone else
